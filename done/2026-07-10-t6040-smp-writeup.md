# m1n1 SMP execute-and-return working on M4 Pro (t6040 "Brava Chop")

*Draft for #asahi-dev — 2026-07-10. Review before posting.*

Tethered t6040 (Mac16,8 / J614s, chipid 0x6040) over USB from an M1 host, m1n1
proxy at EL2, boot core is a P-core (smp_id 4). All 14 cores enumerated and
started (4E + 2×5P; non-contiguous ADT id leaves a gap at smp_id 9).
`CPU_START_OFF_T6031` (0x88000) is correct for Brava.

**broken_wfi on M4:** wfi/wfit lose architectural state on M4 secondaries, so
they park in `wfe` instead of `deep_wfi()`. Local `broken_wfi` flag on
`features_m4` + proxyclient CPUFeatures parse (`broken_wfi=True` confirmed live
on the patched binary).

**Execute-and-return proof:** uploaded an 8-byte leaf `add x0,x0,x1; ret` to
heap (dc_cvau/ic_ivau for I-side coherence), on-target readback verified, then
`smp_call_sync` on an E-core and a P-core secondary with two distinct arg pairs
— `dead0000+0000beef→deadbeef`, `11110000+00002222→11112222`, both correct. So
parked-in-wfe secondaries wake on IPI, execute freshly-uploaded code, and return
values cleanly. (fast_ipi=True.)

**Per-core MPIDR** (`mrs x0, mpidr_el1` leaf, all 14 cores):

```
E-cluster   smp_id 0-3    MPIDR 0x8000000{0..3}   Aff2=0 Aff1=0 Aff0=0-3
P-cluster0  smp_id 4-8    MPIDR 0x8001010{0..4}   Aff2=1 Aff1=1 Aff0=0-4  (4=boot)
P-cluster1  smp_id 10-14  MPIDR 0x8001020{0..4}   Aff2=1 Aff1=2 Aff0=0-4
```

Affinity scheme: Aff2 = E(0)/P(1) domain, Aff1 = cluster, Aff0 = core-in-cluster;
bit31 RES1. smp_id 9 gap lands exactly on the P-cl0→P-cl1 ADT boundary.

**Heartbeat:** all 14 cores executed a cpu-tagged leaf and returned the expected
per-core value; all 13 secondaries `smp_is_alive=1`.

## Session method notes (not for posting)

- No `build/m1n1.elf` in the tree and local rust toolchain broken (hb rustc 1.82
  dyld-fails vs LLVM 22; rustup default 1.58.1; no bare-metal target). Rebuild
  would not reliably byte-match the running image, so symbol addresses from a
  rebuild couldn't be trusted.
- Instead used the proxyclient `ARMAsm` path: compile a tiny deterministic leaf,
  `u.malloc` a heap buffer, `iface.writemem`, `dc_cvau`/`ic_ivau`, then
  `p.smp_call_sync(cpu, addr, ...)`. Boot core (smp_id 4) can't smp_call itself —
  read it via `u.exec("mrs x0, mpidr_el1")`.
- `u.mrs("mpidr_el1")` fails (name not in proxyclient sysreg table); clang's
  assembler encodes `mrs x0, mpidr_el1` (0xd53800a0) directly in the leaf.
